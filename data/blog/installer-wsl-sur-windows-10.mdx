Microsoft et Cannonocal se sont associés pour créer WSL.
Il s'agit d'un sous-système Linux basé sur Ubuntu et tournant sous Windows 10.

# Prérequis

## Windows Creators Update

L'installation de WSL nécessite au préalable de disposer de la Creators Update de Windows 10, sortie en Avril 2017.

Si vous ne disposez pas encore de cette mise à jour, vous pouvez l'installer via l'[assistant mise à jour de Windows 10](https://support.microsoft.com/fr-fr/help/3159635/windows-10-update-assistant).

## Mode développeur

Ouvrez les Paramètres Windows via le raccourci `Win + i`.

Commencez par ouvrir les paramètres de "Mise à jour et sécurité" via les paramères Windows.

![Paramètres Windows](../static/images/2017/05/windows_10_update_config.png)

Puis activez le "Mode Développeur" dans la section "Pour les développeurs".

![Mode développeur](../static/images/2017/05/windows_10_developer.png)

Le mode développeur sera alors téléchargé et activé.

Redémarrez Windows.

# Installer le sous Système Windows pour Linux

Dans le menu démarrer, recherchez "fonctionnalités" et lancez "Activer ou désactiver des fonctionnalités Windows".

![Programmes et fonctionnalités](../static/images/2017/05/windows_10_add_remove_menu.png)

Sélectionnez "Sous-système Windows pour Linux".

![Programmes et fonctionnalités](../static/images/2017/05/windows_10_add_remove_wsl.png)

Redémarrez Windows.

Lancez l'application "bash ubuntu" via le menu démarrer et suivez les instructions.

![Programmes et fonctionnalités](../static/images/2017/05/windows_10_bash_install.png)

Le sous système est installé dans le dossier `C:\Users\[utilisateur]\AppData\Local\lxss`.
**Attention cependant, le contenu de ce dossier ne doit pas être modifié depuis Windows.** Ceci endommagerait les métadonnées des fichiers et les rendrait inutilisables sous WSL.

Il faut éviter de stocker des fichiers directement dans le répertoire home du sous-système. Les fichiers du sous-système n'étant pas conservés lors d'une mise à jour majeur (ex Ubuntu 14.04 à 16.04).
Si vous avez des fichiers à stocker, le mieux est encore de les stocker sur vos disques Windows via les montages /mnt/c, /mnt/d, ...
